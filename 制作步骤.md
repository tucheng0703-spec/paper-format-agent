## 一、项目背景

### 痛点
1. 期刊格式标准不统一，换刊投稿每次都要重新改格式
2. 部分期刊无明确格式要求，只能照着已发表论文"照猫画虎"
3. 改格式是纯体力活，机械但容易出错
4. 图表位置调整困难

### 产品定位
帮助科研人员快速完成论文格式调整的AI智能体，核心差异化：**样例参照改版 + 图表智能排版**

---

## 二、技术方案选型

采用 **LLM + 规则引擎** 方案：

| 组件 | 职责 | 技术选型 |
|------|------|---------|
| 前端 | 用户交互界面 | Streamlit |
| LLM | 理解格式要求、逆向提取格式规则 | 硅基流动 API（Qwen2.5-72B） |
| 规则引擎 | 精确执行Word格式操作 | python-docx |
| PDF解析 | 读取样例论文PDF | pdfplumber |
| 存储 | 历史记录 | SQLite |
| 部署 | 内网穿透暴露公网 | localtunnel |

**为什么选LLM+规则引擎？**
- 纯LLM方案：格式精准度不够，容易改错
- LLM+模板引擎：依赖模板覆盖度，无法处理无模板期刊
- LLM+规则引擎：平衡精准度与灵活性，支持样例逆向提取

## 四、核心模块说明

### 4.1 format_engine.py — 格式规则引擎

定义 `FormatRule` 数据结构，每条规则描述一个论文部分的格式要求：

```python
@dataclass
class FormatRule:
    target: str          # "title" / "abstract" / "body" / "reference" / "figure" / "table"
    font_name: str       # 字体（如"宋体"）
    font_size: float     # 字号（磅）
    bold: bool           # 是否加粗
    italic: bool         # 是否斜体
    alignment: str       # 对齐方式 left/center/justify
    line_spacing: float  # 行距
    first_line_indent: float  # 首行缩进
    space_before: float  # 段前间距
    space_after: float   # 段后间距
```

规则集合 `RuleSet` 支持规则合并、验证、冲突检测。

### 4.2 docx_parser.py — Word文档解析

读取论文的格式信息：
- 每个段落的文字内容、样式、字体、字号
- 加粗/斜体/对齐方式/行距/缩进
- 表格和图片信息
- 输出结构化的文档对象，供LLM分析

### 4.3 docx_writer.py — Word文档生成

接收原始论文 + FormatRule列表：
- 识别论文各部分（标题、摘要、正文、参考文献）
- 逐段应用格式规则
- **关键约束：只改格式属性，不改变文字内容**

### 4.4 llm.py — LLM调用

两个核心Prompt：

**Prompt 1：格式要求理解**
- 输入：用户粘贴的格式要求文本
- 输出：结构化的FormatRule列表（JSON）

**Prompt 2：样例论文格式逆向提取**
- 输入：样例论文各部分的格式信息（从docx_parser提取）
- 输出：结构化的FormatRule列表

### 4.5 pdf_parser.py — PDF解析

解析样例论文PDF，提取文本和基本格式信息，供LLM分析。

## 五、制作步骤（从零开始复现）

### Step 1：环境准备

```bash
# 创建项目目录
mkdir 论文格式智能体 && cd 论文格式智能体

# 创建Python虚拟环境（推荐）
conda create -n paper_format python=3.11
conda activate paper_format
```

### Step 2：安装依赖

```bash
pip install streamlit python-docx openai pdfplumber python-dotenv sqlalchemy pyngrok
```

requirements.txt 内容：
```
streamlit>=1.28.0
python-docx>=0.8.11
openai>=1.10.0
pdfplumber>=0.10.0
python-dotenv>=1.0.0
sqlalchemy>=2.0.0
```

### Step 3：配置环境变量

```bash
# 复制模板
cp .env.example .env

# 编辑.env，填入API Key
# SILICONFLOW_API_KEY=你的API Key
```

### Step 4：开发核心模块

按以下顺序开发：

1. **config.py** — 全局配置（API地址、模型名称、文件路径）
2. **utils/format_engine.py** — FormatRule数据结构 + RuleSet
3. **utils/docx_parser.py** — Word文档格式解析
4. **utils/docx_writer.py** — Word文档格式应用
5. **utils/llm.py** — LLM调用封装（格式理解 + 样例提取）
6. **utils/pdf_parser.py** — PDF解析
7. **utils/storage.py** — SQLite存储
8. **pages/*.py** — 各页面
9. **app.py** — Streamlit主入口

### Step 5：本地启动测试

```bash
streamlit run app.py
# 浏览器打开 http://localhost:8501
```

### Step 6：部署为网页版（公网访问）

```bash
# 后台启动Streamlit
streamlit run app.py --server.port 8501 --server.headless true &

# 安装localtunnel
npm install -g localtunnel

# 暴露公网
lt --port 8501
# 会输出一个公网URL，如 https://xxxx.loca.lt
```

或者用ngrok（更稳定，但需要authtoken）：
```bash
pip install pyngrok
ngrok authtoken 你的token
ngrok http 8501
```

## 六、使用说明

### 模式一：格式模板改版

1. 打开网页 → 选择「格式模板改版」
2. 上传你的论文（.docx）
3. 在文本框粘贴目标期刊的格式要求
4. 点击「开始改版」
5. 等待LLM解析格式 → 规则引擎应用 → 生成新文档
6. 预览修改统计，下载改版后的docx

### 模式二：样例参照改版

1. 打开网页 → 选择「样例参照改版」
2. 上传你的论文（.docx）
3. 上传1-3篇该期刊已发表论文（.docx 或 .pdf）
4. 点击「开始改版」
5. 系统会逆向提取样例论文的格式规则并展示
6. 确认后应用，下载改版后的docx

### 历史记录

所有改版任务自动保存，可在「历史记录」页查看、下载或删除。

## 七、注意事项

1. **API Key必须配置**，否则LLM功能无法使用
2. 仅支持 .docx 格式论文上传，样例论文支持 .docx 和 .pdf
3. 格式修改只改格式属性，**不改变文字内容**
4. 中文字体处理可能有差异，建议改版后人工检查一遍
5. localtunnel的URL是临时的，重启服务后会变化
6. 如需稳定访问，建议使用ngrok或部署到Streamlit Cloud
